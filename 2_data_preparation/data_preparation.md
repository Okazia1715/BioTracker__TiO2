# Data Preparation

This folder contains Python scripts used to clean and prepare datasets
for later analysis and modeling stages.

## Scripts

### 1) Image inversion + contrast adjustment

**Purpose:**  
Prepare histology images for downstream segmentation by applying:

- intensity inversion
- contrast scaling to 1.2

This preprocessing improves Cellpose performance for the next project
stage: training a model to segment adipocytes.

**Input dataset:**

- `1_datasets/Lung_and_Colon_Cancer/Colon_Benign_Tissue/invert/raw/`  
  (raw images, preserved in their original form)

**Processing steps:**

1. Each image was loaded from the input directory
2. Pixel intensities were inverted (bright ↔ dark)
3. A contrast scaling factor of 1.2 was applied
4. The processed images were saved as a new dataset

**Output dataset created:**

- `1_datasets/Lung_and_Colon_Cancer/Colon_Benign_Tissue/invert/processed/`

**Script file:**

- `2_data_preparation/scripts/invert_contrast_1p2.py`

**Notes:**

- The raw dataset in `1_datasets/` was not modified; all preprocessing
  results were written to a separate output directory.
- The same outputs can be fully regenerated by re-running the
  script with the same inputs.

## Data Availability Notice

Due to GitHub repository size limitations, only a small subset of the
processed images is included in this repository.

Specifically, only the first **10 images** of the prepared dataset are
stored locally for demonstration and reproducibility purposes.

These example files allow users to inspect the preprocessing results
and verify the workflow without requiring the full dataset.

The complete set of input images used in this preprocessing step can
be downloaded from Kaggle:

[Colon Microscopy Dataset for Segmentation (Kaggle)](https://www.kaggle.com/datasets/alonaniechvieieva/colon-microscopy-dataset-for-segmentation)

After downloading the dataset, place the images into the input
directory described above:

`1_datasets/colon_benign_tissue/`

The full preprocessing dataset can then be reproduced by running the
provided script:

`2_data_preparation/scripts/invert_contrast_1p2.py`

This approach ensures that the repository remains within GitHub size
limits while still allowing the entire preprocessing workflow to be
fully reproduced.
